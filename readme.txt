== Description ==

**Why use "Kaya Login Captcha"?**

This plugin Adds a simple captcha on login form, register form and lost-password form.

Easy install and use, captcha settings are fully customizable and you can choose the forms on which to display it. The blocked request HTTP status can be customized and the XML-RPC feature can be disabled.

Captcha statistics are also available on the settings page, with the count of passed and blocked requests sorted by year and month.

= Features =

* Captcha available on the login form (Dashboard and WooCommerce).
* Captcha available on the lost-password form (Dashboard and WooCommerce).
* Captcha available on the register form (Dashboard and WooCommerce).
* Editable Captcha code length.
* Editable Captcha code format: numeric, alphabetic or alphanumeric.
* Random lines available in the background of the Captcha.
* Editable blocked request HTTP status.
* XML-RPC WordPress API deactivatable.
* Captcha statistics of passed and blocked requests sorted by year and month.
* Compatible with WordPress MultiSite and WooCommerce.

"Kaya Login Captcha" is a professional login captcha system with fully customizable settings.

= Privacy =

This plugin does not collect or store any user data. It does not set any cookies and does not connect to any third-party applications. This plugin only generate a captcha code to verify human action for selected forms on your settings.

== Frequently Asked Questions ==

= What if I wish to change the captcha display? =

No problem, Kaya Login Captcha is fully customizable.
Change the settings in "Kaya Studio > Login Captcha".

You can enable or disable the forms on which to display it (the login form, the register form and the lost-password form).

You can custom the Captcha code length, code format (numeric, alphabetic or alphanumeric) and enable random lines in the background of the Captcha.

= What if I wish to edit the captcha error HTTP status? =

No problem, Kaya Login Captcha is fully customizable.
Change the settings in "Kaya Studio > Login Captcha".

You can modify the blocked request HTTP status by error code as Unauthorized (code: 401), Forbidden (code: 403) or let it without change (code: 200).

= What if I wish to deactivate the XML-RPC WordPress API? =

No problem, Kaya Login Captcha is fully customizable.
Change the settings in "Kaya Studio > Login Captcha".

You can deactivate the XML-RPC WordPress API. Warning: Some plugins need this, like the Jetpack plugin.

== Installation ==

The quickest way:

1. Go to the Plugins Menu in WordPress and select Plugins > Add new
1. Search for "Kaya Login Captcha"
1. Click "Install" and "activate".

The other way:

1. Upload the "kaya-login-captcha" folder to the "/wp-content/plugins/" directory
1. Activate the plugin through the "Plugins" menu in WordPress.
